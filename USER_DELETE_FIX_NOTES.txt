User Delete Foreign Key Fix
===========================

Fixes the Render error:
psycopg2.errors.ForeignKeyViolation: update or delete on table "users" violates foreign key constraint "lapsed_policies_assigned_agent_id_fkey"

Change made:
- Permanent user delete now looks up every PostgreSQL foreign key that references users.id.
- Before deleting the user, all linked records are reassigned to protected Super Admin wjm@martinsdirect.com.
- This includes lapsed_policies.assigned_agent_id and any future tables that reference users.id.
- Super Admin and Admin protection remains active.

Test:
1. Deploy this package on Render test.
2. Log in as wjm@martinsdirect.com.
3. Delete a normal Agent user with assigned lapsed policies.
4. Confirm the user is deleted and lapsed policy history still exists under Super Admin.
